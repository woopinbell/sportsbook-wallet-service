# 정합성 테스트 (invariant 리스너 + reconciliation drift)

- 해시: `6ad1ca5` test(integrity): cover invariant listener and reconciliation drift detection
- 인위적 drift를 주입해 invariant 리스너와 reconciliation 배치가 잡아내는지 검증한다

## 개요

`IntegrityIntegrationTest`가 Testcontainers PostgreSQL + Redis 위에서 두 정합성 장치를 검증한다. 고의로 깨진 데이터(짝 없는 차변)를 주입하고 Micrometer 카운터가 오르는지 본다. Kafka는 제외한다. 리스너도 배치도 Kafka를 건드리지 않고, 빼면 몇 초 절약된다.

## 작업 순서

1. 정상 debit이 검사 카운터만 올리고 위반은 안 올림을 본다.
2. 짝 없는 차변을 주입하고 `InvariantChecker.verify()`를 직접 호출한다.
3. 같은 drift를 둔 채 `reconcile()`을 돌려 위반 카운터를 본다.

## 작업 내역

### 세 테스트

1. 정상 debit이 `wallet.invariant.checks`를 올리되 `wallet.invariant.violations`는 기준선 그대로. `@TransactionalEventListener(AFTER_COMMIT)` 배선을 end-to-end로 탄다. `debit()`이 `writePair()`에서 `OperationCommitted`를 publish하고 리스너가 커밋 후 fire한다.
2. 짝 없는 차변이 `InvariantChecker`를 트립한다. `injectOrphanDebit()`이 합성 operation_group_id로 차변 한 행만 쓰고(대변 없음), 테스트가 `InvariantChecker.verify()`를 직접 호출한다.
3. 같은 drift로 `reconcile()`을 돌리면 위반 두 개가 등록된다. 시스템 net != 0 검사 하나, 계좌별 스냅샷 대조 하나.

### 직접 호출의 이유

```java
// AFTER_COMMIT 배선은 위 정상 debit 테스트가 이미 증명한다.
// 합성 그룹은 감싸는 트랜잭션이 없어 리스너가 디스패치하지 않으므로 verify()를 직접 부른다.
invariantChecker.verify(new OperationCommitted(syntheticGroup));
```

이 결정은 실제로 한 번 빨갛게 뜬 뒤에 내렸다. 처음엔 `publishEvent`로 합성 이벤트를 쐈는데 AFTER_COMMIT 리스너가 감싸는 트랜잭션이 없으면 디스패치 자체를 안 해서 카운터가 0이었다. 정상 경로 테스트가 배선을 증명하므로, drift 테스트는 검증 로직을 직접 부르는 쪽이 맞다.

### 합성 행을 스키마 적합하게

`injectOrphanDebit`은 `LedgerEntry.pair()`로 유효한 쌍을 만든 뒤 차변 절반만 저장한다. 합성 행이 NOT NULL과 CHECK 제약을 만족하므로, DB의 국소 sanity 검사를 우회하지 않고 상위 invariant만 시험한다.

## 결과

스위트 전체 46개가 통과했다. 두 정합성 장치가 실제 drift를 잡아 메트릭으로 노출함을 확인했다.

## 요약

- 짝 없는 차변을 주입해 리스너와 배치가 위반을 카운트하는지 본다.
- AFTER_COMMIT 배선은 정상 debit 테스트로, 검증 로직은 직접 호출로 나눠 검증한다.
- 합성 행을 스키마 적합하게 만들어 상위 invariant만 시험한다.

## 핵심 확인

- 테스트 진입점: `IntegrityIntegrationTest`.
- AFTER_COMMIT 리스너는 감싸는 트랜잭션이 없으면 디스패치하지 않는다(직접 호출 이유).
- reconciliation은 한 drift에 두 위반(net + 스냅샷)을 센다.

## 기억/설명 Level

### L2 - 코드/문서 기반 설명 대상

- AFTER_COMMIT 배선 검증과 검증 로직 검증을 나눈 이유(리스너가 트랜잭션 밖에서 디스패치 안 함)
- `injectOrphanDebit`이 스키마 적합 행으로 상위 invariant만 시험하는 방식

### L1 - 읽고 파악하면 충분한 대상

- 한 drift가 reconciliation에서 두 위반으로 집계되는 점
