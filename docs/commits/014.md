# RFC 7807 ProblemDetail 예외 매핑

- 해시: `445e92c` feat(api): add RFC 7807 ProblemDetail mapping for domain exceptions
- 도메인/프레임워크 예외를 안정적인 status + 기계 판독 가능한 code slug로 번역하는 ControllerAdvice를 둔다

## 개요

`GlobalExceptionHandler`가 wallet REST 표면의 HTTP 오류 응답을 한곳에 모은다(ADR-0004). `@RestControllerAdvice`는 컨트롤러 전역에서 예외를 가로채는 Spring 메커니즘이다. 각 도메인 예외 타입이 안정적인 status와 기계 판독용 `code` slug로 매핑돼, 호출자가 산문을 파싱하지 않고 slug로 분기할 수 있다.

## 작업 순서

1. 도메인 예외 4종을 status + slug로 매핑한다.
2. 프레임워크 예외(헤더 누락, 잘못된 JSON 등)를 400으로 모은다.
3. 컨텍스트 속성을 ProblemDetail에 붙인다.
4. 500 catch-all은 응답 본문을 좁게 유지한다.

## 작업 내역

### status / slug 표

| Status | Slug | 트리거 |
|--------|------|--------|
| 400 | WALLET_BAD_REQUEST | IllegalArgumentException, 헤더 누락, 잘못된 JSON/UUID, IdempotencyKey 형태 위반 |
| 400 | WALLET_VALIDATION_FAILED | `@Valid` 본문의 Bean Validation 위반 |
| 404 | WALLET_ACCOUNT_NOT_FOUND | AccountNotFoundException |
| 409 | WALLET_IDEMPOTENCY_CONFLICT | IdempotencyConflictException |
| 422 | WALLET_INSUFFICIENT_BALANCE | InsufficientBalanceException |
| 422 | WALLET_CURRENCY_MISMATCH | CurrencyMismatchException |
| 500 | WALLET_INTERNAL_ERROR | 그 외 전부, ERROR 로그 |

```java
@ExceptionHandler(InsufficientBalanceException.class)
public ProblemDetail insufficientBalance(InsufficientBalanceException e) {
  ProblemDetail problem = ProblemDetail.forStatusAndDetail(HttpStatus.UNPROCESSABLE_ENTITY, e.getMessage());
  problem.setProperty("code", "WALLET_INSUFFICIENT_BALANCE");
  problem.setProperty("requested", e.requested());
  problem.setProperty("available", e.available());
  return problem;
}
```

각 ProblemDetail은 컨텍스트 속성을 실어 보낸다. InsufficientBalance의 requested/available 금액, CurrencyMismatch의 expected/actual 통화, IdempotencyConflict의 키와 충돌 사유다. 호출자가 추가 probe 호출 없이 실패에 대응할 수 있다.

Spring 6은 컨트롤러가 `ProblemDetail`을 반환하면 `application/problem+json` 콘텐츠 타입을 자동으로 붙인다. 수동 `ResponseEntity` 배선이 없어 핸들러가 평탄하다.

### 500 catch-all

```java
@ExceptionHandler(Exception.class)
public ProblemDetail internalError(Exception e) {
  log.error("Unhandled exception bubbled up to GlobalExceptionHandler", e);
  ProblemDetail problem = ProblemDetail.forStatusAndDetail(
      HttpStatus.INTERNAL_SERVER_ERROR, "Unexpected error — see server logs for trace");
  problem.setProperty("code", "WALLET_INTERNAL_ERROR");
  return problem;
}
```

500의 응답 detail을 일부러 좁게 둬서 예외 메시지가 내부 상태를 응답 본문으로 흘리지 못하게 한다. 전체 trace는 JSON 구조화 로그에 ERROR로 남는다(ADR-0007).

## 결과

호출자는 `code` slug 하나로 모든 실패를 분기할 수 있고, 모든 오류 응답이 `application/problem+json`이라 콘텐츠 타입만으로 라우팅 가능하다.

## 요약

- 도메인 예외 6종 + 프레임워크 예외를 status + `code` slug로 일관 매핑한다.
- ProblemDetail에 컨텍스트 속성을 실어 추가 probe 없이 대응 가능하게 한다.
- Spring 6이 `application/problem+json`을 자동 부여한다.
- 500은 본문을 좁게 둬 내부 상태 누출을 막고 trace는 로그로 보낸다.

## 핵심 확인

- 호출자는 `code` slug로 분기한다. status만으로는 422 두 종(잔고/통화)을 구분 못 한다.
- 500 응답 본문에는 예외 메시지를 넣지 않는다.
