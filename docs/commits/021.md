# maven wrapper 체크인 (균일 빌드)

- 해시: `28e681f` build(maven): check in maven wrapper for uniform builds
- risk-service의 maven wrapper 3파일을 동일본으로 체크인해 orchestration `build-all.sh --rebuild`의 `./mvnw` 가정을 충족시킨다 — **phase 2**(후속 윈도우, commit-policy phase 모델)의 커밋, 코드 변경 없음.

## 개요

orchestration `scripts/build-all.sh --rebuild`는 각 서비스 디렉터리에서 `./mvnw -q -DskipTests package`를 호출하는데, maven wrapper는 risk-service·odds-feed-service·shared-protocol 3곳에만 체크인돼 있었다. wallet을 포함한 나머지 5개 서비스에서는 `--rebuild`가 "no such file"로 실패한다 — 4차 품질 런이 §7(소스 결함, 보고만)로 박제한 결함이다(orchestration 003 답지 `## 알려진 소스 결함`). orchestration phase 2(`519aad8`, 010 답지)가 시스템 `mvn` 폴백으로 빌드를 살리고 래퍼 균일화를 각 서비스의 후속 phase 후보로 남겼고, 이 커밋이 wallet 몫을 채운다.

## 작업 순서

1. risk-service에서 `mvnw`·`mvnw.cmd`·`.mvn/wrapper/maven-wrapper.properties` 복사, `chmod +x mvnw`.
2. `diff -r`로 byte-identical 확인.
3. `./mvnw -v`로 래퍼 동작 검증.

## 작업 내역

`.mvn/wrapper/maven-wrapper.properties`는 `wrapperVersion=3.3.4`, `distributionType=only-script`, Maven `3.9.11` — jar 없이 `mvnw` 스크립트가 배포본을 직접 받아 `~/.m2/wrapper/dists`에 캐시하는 구성이다. 새로 쓰지 않고 risk-service 것을 그대로 가져왔다: 래퍼 버전이 서비스마다 갈리면 균일화가 아니라 또 다른 드리프트다. 체크인 후 세 파일의 git blob 해시가 risk-service와 일치함을 확인했다. scaffold 때부터 `.gitignore`가 `.mvn/wrapper/maven-wrapper.jar`·`.mvn/timing.properties`를 제외하고 있어(래퍼를 전제한 ignore) 충돌 없음.

## 결과

깨끗한 체크아웃에서 `build-all.sh --rebuild`가 wallet에 대해 시스템 mvn 폴백 없이 `./mvnw`로 동작한다.

## 요약

- risk-service의 maven wrapper(3.3.4, only-script, Maven 3.9.11)를 동일본으로 체크인.
- 코드 변경 없음 — §7 보고(래퍼 불균일)의 wallet 몫 해소.

## 다음 작업

wallet 측 follow-up 없음. 같은 phase에서 gateway·betting·admin-api·settlement도 동일본을 체크인한다.
