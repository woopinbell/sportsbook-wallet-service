# OutboxEvent entity + 스케줄 publisher

outbox 행 entity와 1초 주기로 미발행 행을 Kafka로 흘리는 스케줄 publisher, 그리고 Kafka producer 설정을 만든다.

## 개요

앞 커밋이 만든 outbox 테이블에 매핑되는 `OutboxEvent` entity와, 그 테이블을 주기적으로 비우는 `OutboxPublisher`를 도입한다. 잔고를 바꾸는 모든 연산은 `OutboxEvent.pending()`으로 행을 만들어 분개 쌍과 같은 트랜잭션에 저장한다(배선은 011).

## 작업 순서

1. `OutboxEvent` entity를 write-once에 가깝게 만든다(`publishedAt`만 변경 가능).
2. `OutboxEventRepository.findUnpublished(Pageable)`로 배치 조회를 노출한다.
3. `OutboxPublisher`를 `@Scheduled` 1초 주기로 돌린다.
4. `KafkaConfig`로 `(String, byte[])` producer를 wire한다.

## 작업 내역

### `OutboxEvent`

대부분 필드가 write-once다. `publishedAt`만 변경 가능하고, `payload`는 생성·읽기 양쪽에서 방어적으로 clone해 BYTEA 버퍼를 Java 측에서 불변으로 유지한다. 접근자는 계약이라 노출한다.

**책임:** `markPublished`를 멱등으로 만들어 중복 발행 응답이 행을 흔들지 못하게 한다. `pending()` 팩토리는 미발행 행을 만든다.

```java
public class OutboxEvent {
  // write-once 필드들 + publishedAt(변경 가능)
  public static OutboxEvent pending(String topic, String partitionKey, String schemaName, byte[] payload, Instant now) {

  }

}
```

### `OutboxPublisher`

매 tick마다 미발행 행을 오래된 순으로 최대 `BATCH_SIZE`(기본 100)개 가져와 각각 Kafka로 보내 ack를 기다린(5초 timeout) 뒤 `published_at`을 찍는다. `Pageable`로 배치를 제한한 것은 backlog가 스케줄러를 멈추지 못하게 하기 위함이다.

**책임:** 실패하면 로그만 남기고 행을 미발행으로 둬 다음 tick이 재시도한다 — 이 retry-until-acked가 outbox를 at-least-once로 만든다. dirty checking으로 entity 변경이 메서드 종료 시 커밋된다.

```java
@Scheduled(fixedDelayString = "${wallet.outbox.poll-interval-ms:1000}")
@Transactional
public void publishPending() {
  // TODO: repository.findUnpublished(PageRequest.of(0, BATCH_SIZE)) 순회
  //       각 행 sendBlocking(ack 대기) 성공 시 markPublished(clock.instant())
  //       실패 행은 미발행으로 둠(다음 tick 재시도)
}
```

`OutboxEventRepository.findUnpublished(Pageable)`는 `published_at IS NULL` 행을 `created_at` 오름차순으로 돌려준다(partial index 경로).

### `KafkaConfig`

value를 `byte[]`로 둔 것은 outbox 행이 이미 Avro 인코딩된 payload를 들고 있기 때문이다.

**책임:** producer 설정 세 개로, 브로커 내부 재시도가 레코드를 중복시키지 않게 한다(idempotent producer). outbox의 retry-until-acked와 합쳐져 파티션 순서를 보존하는 at-least-once가 된다.

## 검증

- `mvn -q -Dtest=OutboxIntegrationTest test` — publisher가 미발행 행을 embedded 브로커로 흘리고 `published_at`을 찍는 경로(012의 5번 테스트). *Testcontainers + embedded Kafka가 떠야 한다.* (이 커밋 시점엔 `WalletService`가 아직 outbox에 쓰지 않으므로, 동작은 011 배선 + 012 테스트에서 관찰된다.)
- 감각 점검: 발행이 실패하면 행이 미발행으로 남아 다음 tick이 재시도하는가 / `markPublished`를 두 번 불러도 `published_at`이 처음 값으로 고정되는가.
