# Flyway V1 스키마 (account + ledger_entry)

앞선 두 도메인 커밋의 JPA 매핑을 PostgreSQL 테이블로 물질화하고 제약과 인덱스를 SQL로 박는다.

## 개요

Flyway는 SQL 파일 기반 마이그레이션 도구다. `V<버전>__<설명>.sql` 파일을 버전 순으로 적용하고, 애플리케이션 시작 시 자동으로 미적용 마이그레이션을 검증한다. 스키마 변경 이력 자체가 코드 저장소에 남고 PR 리뷰 대상이 된다.

이 커밋은 `account`와 `ledger_entry` 두 테이블을 만든다. 두 테이블 사이에 외래 키(FK)는 두지 않는다.

## 작업 순서

1. `account` 테이블에 두 bucket 컬럼과 통화/음수 방지 CHECK 제약을 둔다.
2. `ledger_entry` 테이블에 멱등성 복합 unique 제약 두 개를 둔다.
3. 읽기 경로용 인덱스 두 개를 만든다.
4. 모든 TIMESTAMP를 WITH TIME ZONE으로 둔다(ADR-0003).

## 작업 내역

> 컬럼 골격은 노출한다(매핑 계약). 제약·인덱스 정의는 비운다 — 거기 설계가 있다.

### `account` 테이블

CHECK 제약은 도메인 invariant를 DB에 한 번 더 박는다. Hibernate 매핑이 잘못되거나 누군가 직접 UPDATE를 해도 테이블이 도메인이 거부할 상태로 가지 못하게 하는 defence-in-depth다.

```sql
CREATE TABLE account (
    user_id            UUID                     PRIMARY KEY,
    available_amount   BIGINT                   NOT NULL,
    available_currency VARCHAR(3)               NOT NULL,
    locked_amount      BIGINT                   NOT NULL,
    locked_currency    VARCHAR(3)               NOT NULL,
    version            BIGINT                   NOT NULL DEFAULT 0,
    created_at         TIMESTAMP WITH TIME ZONE NOT NULL,
    updated_at         TIMESTAMP WITH TIME ZONE NOT NULL
    -- TODO: CHECK 제약 3개 — 두 bucket 통화 일치, available/locked 각각 음수 금지
);
```

### `ledger_entry` 테이블

**책임:** 복합 unique 두 개로 matched-pair 멱등성을 인코딩한다(002의 `UNIQUE(idempotency_key)` 충돌 해결을 SQL로). bucket/side/reason은 CHECK로 enum 값 집합을 박아 직접 INSERT가 알 수 없는 값을 못 넣고 스키마가 자기 문서화되게 한다.

```sql
CREATE TABLE ledger_entry (
    entry_id           UUID                     PRIMARY KEY,
    account_id         UUID                     NOT NULL,
    bucket             VARCHAR(16)              NOT NULL,
    side               VARCHAR(6)               NOT NULL,
    amount             BIGINT                   NOT NULL,
    currency           VARCHAR(3)               NOT NULL,
    reason             VARCHAR(16)              NOT NULL,
    idempotency_key    VARCHAR(128)             NOT NULL,
    operation_group_id UUID                     NOT NULL,
    created_at         TIMESTAMP WITH TIME ZONE NOT NULL
    -- TODO: amount > 0 CHECK
    -- TODO: bucket / side / reason 각각 enum 값 집합 CHECK
    -- TODO: matched-pair 멱등성 복합 unique 두 개 — 호출 키용 하나 + 내부 그룹 ID용 방어 하나.
    --       같은 키 재시도는 막되 한 쌍의 두 행은 양립해야 한다(002의 충돌 해결을 SQL로 — 키에 무엇을 더하나?)
);
```

`account_id`에서 `account.user_id`로 가는 FK는 두지 않는다. ledger는 시스템 계좌(HOUSE, EXTERNAL_PAYMENT)도 상대로 쓰는데 이들은 account 테이블에 행이 없기 때문이다. account_id 유효성은 service 레이어가 보장한다.
