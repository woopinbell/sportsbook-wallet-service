# DailyReconciliationJob (ledger drift 탐지)

매일 새벽 전체 ledger 합과 계좌별 스냅샷-원장 일치를 검증하는 배치를 둔다.

## 개요

`DailyReconciliationJob`은 per-operation `InvariantChecker`가 개별 쌍만으로는 볼 수 없는 두 가지를 매일 한 번 점검한다. 시스템 전체 차변−대변 = 0과, 각 계좌 스냅샷이 원장 entry 합과 일치하는지다. 후자가 wallet의 "스냅샷 + 증분" 모델의 닫는 고리다. 스냅샷이 빠른 read 경로이고 ledger entry가 audit ground truth다.

## 작업 순서

1. `@Scheduled(cron)`으로 매일 03:00 UTC에 돈다.
2. 시스템 전체 net을 검사한다.
3. 계좌마다 두 bucket 스냅샷을 원장 합과 대조한다.
4. Micrometer 카운터/게이지로 상태를 노출한다.

## 작업 내역

### 두 검증

스케줄/트랜잭션 어노테이션은 노출한다. 검증 본문은 비운다.

**책임:** per-operation 리스너가 각 쌍을 국소적으로 보는 반면, 이 배치는 연산을 가로질러 나타나는 drift(빠진 entry, 손으로 고친 SQL, 오래된 덤프 복원)를 잡는다. `@Transactional(readOnly = true)`로 순회 중 변경을 막는다.

```java
@Scheduled(cron = "${wallet.reconciliation.cron:0 0 3 * * *}")
@Transactional(readOnly = true)
public void reconcile() {
  // TODO: runs.increment()
  // TODO: net = ledgerRepo.netSumAll(); systemNet 게이지에 반영; net != 0이면 violations + ERROR 로그
  // TODO: 모든 account 순회 — verifyBucket(account, AVAILABLE, available) / (account, LOCKED, locked)
  //       스냅샷 bucket 값과 netByAccountAndBucket 원장 합이 어긋나면 위반
}
```

애플리케이션 타임존이 UTC(ADR-0003)라 기본 cron은 03:00 UTC다. 운영자가 `wallet.reconciliation.cron`으로 덮을 수 있다. 전체 테이블 순회는 V1에서 단순하게 뒀고, `findAll(Pageable)` 페이지네이션은 wallet 테이블이 커질 때의 미래 개선이다.

### Micrometer 메트릭 셋

**책임:** 세 메트릭으로 상태를 Prometheus에 노출한다 — `wallet.reconciliation.runs`(tick 카운터), `wallet.reconciliation.violations`(위반 카운터), `wallet.reconciliation.net`(마지막 관측 net 게이지, 정상 0). net 게이지는 인메모리 `AtomicLong`을 레지스트리에 Supplier로 공유해 scrape마다 마지막 reconciliation 값이 보이게 한다.

## 검증

- `mvn -q -Dtest=IntegrityIntegrationTest test` — 주입한 drift로 `reconcile()`을 돌리면 위반 카운터가 오르는지(018; 한 drift가 net + 스냅샷 두 위반). *Testcontainers가 떠야 한다.*
- 감각 점검: 정상 상태에서 `reconcile()`이 violations를 올리지 *않고* net 게이지가 0인가 / 합성 drift 후 net 게이지가 0이 아닌 값으로 보이는가.
