# RFC 7807 ProblemDetail 예외 매핑

도메인/프레임워크 예외를 안정적인 status + 기계 판독 가능한 code slug로 번역하는 ControllerAdvice를 둔다.

## 개요

`GlobalExceptionHandler`가 wallet REST 표면의 HTTP 오류 응답을 한곳에 모은다(ADR-0004). `@RestControllerAdvice`는 컨트롤러 전역에서 예외를 가로채는 Spring 메커니즘이다. 각 도메인 예외 타입이 안정적인 status와 기계 판독용 `code` slug로 매핑돼, 호출자가 산문을 파싱하지 않고 slug로 분기할 수 있다.

## 작업 순서

1. 도메인 예외 4종을 status + slug로 매핑한다.
2. 프레임워크 예외(헤더 누락, 잘못된 JSON 등)를 400으로 모은다.
3. 컨텍스트 속성을 ProblemDetail에 붙인다.
4. 500 catch-all은 응답 본문을 좁게 유지한다.

## 작업 내역

### status / slug 표

trigger(어떤 예외)는 노출하고, 그 예외에 줄 status와 slug는 비운다 — 거기 매핑 설계가 있다.

| Status | Slug | 트리거 |
|--------|------|--------|
| 400 | WALLET_BAD_REQUEST | IllegalArgumentException, 헤더 누락, 잘못된 JSON/UUID, IdempotencyKey 형태 위반 *(예시)* |
| ? | ? | `@Valid` 본문의 Bean Validation 위반 |
| ? | ? | AccountNotFoundException |
| ? | ? | IdempotencyConflictException |
| ? | ? | InsufficientBalanceException |
| ? | ? | CurrencyMismatchException |
| ? | ? | 그 외 전부 (+ ERROR 로그) |

**책임:** 각 ProblemDetail은 컨텍스트 속성을 실어 보낸다(InsufficientBalance의 requested/available, CurrencyMismatch의 expected/actual, IdempotencyConflict의 키·사유). 호출자가 추가 probe 없이 실패에 대응하게. Spring 6은 컨트롤러가 `ProblemDetail`을 반환하면 `application/problem+json`을 자동 부여하므로 수동 `ResponseEntity` 배선이 없다.

```java
@ExceptionHandler(InsufficientBalanceException.class)
public ProblemDetail insufficientBalance(InsufficientBalanceException e) {
  // TODO: 적절한 status + code slug + requested/available 컨텍스트 속성을 실은 ProblemDetail
}
```

### 500 catch-all

**책임:** 500의 응답 detail을 일부러 좁게 둬 예외 메시지가 내부 상태를 응답 본문으로 흘리지 못하게 한다. 전체 trace는 JSON 구조화 로그에 ERROR로 남긴다(ADR-0007).

```java
@ExceptionHandler(Exception.class)
public ProblemDetail internalError(Exception e) {
  // TODO: 전체 trace를 ERROR 로그로 남기고, 표 규약에 맞는 code slug를 실은 500 ProblemDetail 반환.
  //       detail은 좁게 둬 예외 메시지가 내부 상태를 응답 본문으로 흘리지 않게 한다.
}
```

## 검증

- `mvn -q -Dtest=WalletControllerTest test` — 슬라이스 테스트(015)가 각 예외의 status·slug·`application/problem+json`을 검증한다(Docker 불필요).
- 감각 점검: 잔고 부족과 통화 불일치가 *둘 다 422*인데 `code` slug로 구분되는가(status만으론 불가) / 500 응답 본문에 예외 메시지가 들어가지 *않는가* / 오류 응답 콘텐츠 타입이 `application/problem+json`인가.
