# WalletController 테스트 (@WebMvcTest)

컨트롤러 슬라이스만 띄워 endpoint와 ProblemDetail 응답을 MockMvc로 검증한다.

## 개요

`@WebMvcTest`는 웹 계층만 로드하는 Spring 테스트 슬라이스다. `WalletController`와 `GlobalExceptionHandler`만 띄우고 `WalletService`는 `@MockBean`으로 대체한다. PostgreSQL/Redis/Kafka 컨테이너가 필요 없어 약 2초에 끝난다.

## 작업 순서

1. `@WebMvcTest(WalletController.class)` + `@Import(GlobalExceptionHandler.class)`로 슬라이스를 잡는다.
2. `WalletService`를 `@MockBean`으로 stub한다.
3. endpoint별 200/4xx 케이스를 MockMvc로 검증한다.

## 작업 내역

### 슬라이스 구성

슬라이스 배선은 계약이라 노출한다.

```java
@WebMvcTest(WalletController.class)
@Import(GlobalExceptionHandler.class)
class WalletControllerTest {
  @Autowired MockMvc mvc;
  @MockBean WalletService walletService;
}
```

`MockMvc`는 실제 서블릿 컨테이너 없이 컨트롤러에 요청을 보내는 도구다. `WalletService`가 mock이라 컨트롤러의 매핑/검증/직렬화만 본다.

### 열두 테스트

케이스 의도와 기대 status/slug는 노출한다. stub 설정과 요청/단언 본문은 비운다.

- openAccount(2): 200 계좌 스냅샷, `@NotNull` 위반 시 400 WALLET_VALIDATION_FAILED.
- getBalance(2): 200 available/locked/total, 계좌 없으면 404 WALLET_ACCOUNT_NOT_FOUND(userId 실음).
- deposit(2): 200 operationGroupId+reason, `Idempotency-Key` 헤더 누락 시 400(detail에 "Idempotency-Key").
- debit(3): 422 INSUFFICIENT_BALANCE(requested/available), 422 CURRENCY_MISMATCH(expected/actual), 409 IDEMPOTENCY_CONFLICT(키+사유).
- credit(1): source=HOUSE_POOL → reason=BET_PAYOUT.
- 잘못된 입력(2): 비 JSON 400, MAX_LENGTH 초과 Idempotency-Key 400.

**책임:** 모든 ProblemDetail 응답이 `application/problem+json`임을 단언해, downstream consumer가 콘텐츠 타입만으로 라우팅 가능함을 고정한다.

```java
mvc.perform(post("/internal/v1/wallet/transactions/debit")
        .header("Idempotency-Key", "bet-1")
        .content(/* TODO: userId + amount JSON */))
    // TODO: status 422 + contentType problem+json + jsonPath("$.code") == "WALLET_INSUFFICIENT_BALANCE"
    ;
```

## 검증

- `mvn -q -Dtest=WalletControllerTest test` — 열두 테스트 green. service가 mock이라 DB/Redis/Kafka·Docker가 필요 없고 ~2초에 끝난다.
- 감각 점검: `WalletService` stub이 예외를 던지도록 설정하면 그 예외가 기대한 status/slug로 번역되는가 / 비운 단언을 채우기 전엔 빨강인가.
