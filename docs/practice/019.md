# 부하/증명 테스트 baseline (k6)

k6 하니스와 첫 dev-host baseline을 박제한다. 처리량 + Level 3 정합성 증명.

## 개요

`load-test/` 디렉터리에 k6 하니스와 첫 큐레이션 baseline을 추가한다. k6는 JavaScript로 시나리오를 짜는 부하 테스트 도구다. docker-compose가 PostgreSQL + Redis + Kafka + wallet JAR를 비표준 포트로 띄워 로컬 dev DB와 충돌하지 않게 한다.

## 작업 순서

1. docker-compose로 격리된 부하 테스트 스택을 띄운다.
2. `seed.js`로 계좌를 만들고 큰 금액을 입금한다.
3. `debit_load.js`로 constant-arrival-rate 처리량을 측정한다.
4. `concurrency.js` / `idempotency.js`로 Level 3 정합성을 증명한다.
5. 결과를 `results/<날짜>/`와 `BEST.md`에 박제한다.

## 작업 내역

### 하니스 구성

디렉터리 골격은 노출한다. 격리 포트와 k6 threshold 값은 비운다 — 거기 CI 게이트 설계가 있다.

```
load-test/
  docker-compose.yml   격리 포트 스택. apache/kafka 3.7 KRaft 모드 (arm64 manifest)
  scenarios/
    seed.js            계좌 N개 + 큰 금액 입금
    debit_load.js      constant-arrival-rate, threshold 내장
    concurrency.js     한 user 100 동시 debit, distinct key
    idempotency.js     같은 Idempotency-Key 100 동시 debit
  results/<date>/      baseline JSON
  results/BEST.md      큐레이션 결과 + 환경 메모
```

**책임:** `debit_load.js`의 k6 `thresholds`에 목표를 박아 미달 시 exit code가 0이 아니게 한다(그대로 CI 게이트). 목표 두 개를 재유도하라 — 실패율 상한과 p99 지연 상한은 각각 얼마인가? docker-compose는 로컬 dev DB와 충돌하지 않도록 비표준 포트를 쓴다(어떤 포트 집합?).

### dev-host 측정 (M1 Pro + Docker Desktop 28, 8 CPU)

> 이 단위는 *관찰 결과*다. 재유도 대상이 아니다 — 네 호스트에서 직접 돌려 수치를 기록하고, 답지(`../commits/019.md`)의 dev-host 표(M1 Pro 기준 500 RPS / p95 3.15ms / 에러 0%)와 비교하라. 처리량 한계가 hardware bound인지(설계 한계가 아니라) 판단하는 연습이다.

### Level 3 정합성 증명

두 시나리오는 잔고 결과가 결정적이라 pass/fail로 고정된다. 단언을 비운다.

**책임:**
- `concurrency.js`: 100개 *distinct-key* debit(각 100 KRW) → 그 user의 잔고가 어떻게 변해야 하나? double-spend는 몇 건이어야 하나?
- `idempotency.js`: 100개 *same-key* debit(각 100 KRW) → 몇 번 적용돼야 하나? 나머지는 무엇(Redis fast path / `UNIQUE(idempotency_key, side)`)에 막혀 원래 operation_group_id를 돌려받나?

## 검증

- 스택 기동 후 `k6 run load-test/scenarios/debit_load.js` 가 threshold를 통과(exit 0)하고, `concurrency.js` / `idempotency.js` 의 잔고 단언이 green. *docker-compose 스택(PostgreSQL+Redis+Kafka+wallet JAR)이 떠야 한다.*
- 감각 점검: same-key 100 동시 debit 뒤 그 키의 ledger pair가 정확히 하나인가(008의 정합성 증명을 부하 규모로 재현) / threshold를 일부러 빡세게 낮추면 k6 exit code가 0이 아닌가(CI 게이트로 동작).
