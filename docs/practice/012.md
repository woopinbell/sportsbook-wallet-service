# outbox 통합 테스트 (원자성 + end-to-end 발행)

분개와 outbox가 같은 트랜잭션에 쓰이는지, publisher가 embedded Kafka로 흘려보내는지를 검증한다.

## 개요

`OutboxIntegrationTest`가 transactional outbox를 PostgreSQL + Redis(Testcontainers) + embedded Kafka 브로커(spring-kafka-test) 위에서 검증한다. 다섯 테스트로 발행 형태별 하나씩과 end-to-end drain을 본다.

## 작업 순서

1. `@EmbeddedKafka`로 인메모리 브로커를 띄운다.
2. 자동 발행 스케줄을 24시간으로 밀어 테스트가 발행 시점을 직접 통제한다.
3. 각 연산 후 outbox 행을 읽어 Avro payload를 디코딩해 검증한다.
4. `publishPending()`을 직접 호출해 brokered 발행을 본다.

## 작업 내역

### embedded Kafka 배선

PostgreSQL + Redis 컨테이너와 `@DynamicPropertySource` 주입은 008과 같다. 추가된 것은 `@EmbeddedKafka`로 세 토픽을 미리 만들고 broker 주소를 프로퍼티로 흘리는 부분이다. 토픽 상수는 `WalletEventFactory`에서 가져와 producer와 test가 같은 이름을 공유한다(계약이므로 노출).

```java
@SpringBootTest
@Testcontainers
@EmbeddedKafka(
    partitions = 1,
    topics = {
      WalletEventFactory.TOPIC_DEBITED,
      WalletEventFactory.TOPIC_CREDITED,
      WalletEventFactory.TOPIC_DEBIT_FAILED
    },
    bootstrapServersProperty = "spring.kafka.bootstrap-servers")
@ActiveProfiles("test")
class OutboxIntegrationTest { /* @Container postgres/redis + @DynamicPropertySource: 008과 동일 */ }
```

### 자동 tick 멈추기

**책임:** `OutboxPublisher`의 `@Scheduled` drain을 테스트 중 꺼서, 각 테스트가 `publishPending()`을 직접 불러 발행 시점을 통제하게 한다. park하지 않으면 write와 assertion 사이에 tick이 끼어들어 `published_at`을 미리 찍고, "쓴 직후엔 미발행"이라는 1·4번 검증이 거짓 음성이 된다.

```yaml
wallet:
  outbox:
    poll-interval-ms: # TODO: drain이 테스트 동안 안 돌도록 충분히 큰 값(예: 24h)
```

### 다섯 테스트

케이스 의도만 노출하고 assertion은 비운다.

1. debit 성공 → `WalletDebited` 행이 분개 쌍과 같은 트랜잭션에 남는다(partition_key=userId, `published_at` NULL, payload의 amount/currency/idempotency_key 일치, `ledger_tx_id == operation_group_id`).
2. HOUSE_POOL credit → `WalletCredited` reason=PAYOUT.
3. USER_LOCKED credit → `WalletCredited` reason=REFUND.
4. InsufficientBalance → 메인 트랜잭션이 롤백돼도 `WalletDebitFailed` 행이 reason=INSUFFICIENT_BALANCE로 존재(별도 트랜잭션 덕).

**책임:** 기존 `serialize`의 짝으로 작은 `deserialize`를 추가해, 테스트가 매번 `DatumReader`를 새로 만들지 않고 payload 내용을 검증하게 한다. 역직렬화 측은 현재 테스트만 쓰지만, codec 쌍을 한자리에 둬 downstream consumer가 발견하기 쉽게 한다.

## 검증

- `mvn -q -Dtest=OutboxIntegrationTest test` — 다섯 테스트 green. *Testcontainers(PostgreSQL+Redis) + embedded Kafka가 떠야 한다.*
- 감각 점검: tick park를 빼면 1·4번이 가끔 거짓 음성으로 깨지는가(쓴 직후 `published_at`이 이미 채워짐) / 4번에서 메인 롤백 뒤에도 실패 행이 남는가.
- 종료 시점 Kafka warning은 embedded 브로커가 임시 로그 디렉터리를 정리하며 내는 것으로 테스트 실패가 아니다.
